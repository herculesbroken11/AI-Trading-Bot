# Development Log & Scratchbook

Client request: "Save all your daily logs or history what you have done for developing this trading bot. Take screenshots for test driving the software in the future. One day when you get all done and make a book and capitalize your experience."

## Structure

- `daily/` - Daily development logs (what was done, what broke, what was fixed)
- `screenshots/` - Screenshots from test driving the software
- `decisions/` - "Why we changed this" notes - rationale for design decisions
- `weekly/` - Weekly progress updates for client

## How to Use

1. **Daily Log**: Add a file like `daily/2025-02-27.md` with date and notes
2. **Screenshots**: Save test runs, dashboard views, trade results with descriptive names
3. **Decisions**: Document significant changes and why (e.g. `decisions/2025-02-capital-management.md`)
4. **Weekly**: Use template in `weekly/TEMPLATE.md` for client updates
